# Assets And Files Contract

## Owner

- Primary: tooling-agent
- Supporting: canvas-agent, pm-agent
- Shared source: `shared/assets.ts`, `shared/nodes.ts`

## Scope

This contract covers generated assets, imported local files, asset folders, image categories/tags, reference integrity, safe renderer URLs, cloud URL metadata, and local file library behavior. Assets live under app-controlled storage; DB records store relative paths and optional normalized cloud URL metadata only.

Non-goals:

- No absolute filesystem paths in renderer responses.
- No provider temporary URLs as durable asset references.
- No Seedance, real-person, or liveness-auth asset system in the local version.
- No separate role/scene/prop/creature entity tables; those concepts are image asset categories.

### `asset.rename`

Request:

```ts
interface AssetRenameRequest {
  assetId: string
  displayName: string
}
```

Response:

```ts

```

`asset.rename` updates the logical display label only. It SHALL NOT rename
local files, cloud objects, `relativePath`, `safeUrl`, or `s3Key`; this keeps
canvas/node references stable after a user-visible rename.

Rules:

- Generated bytes SHALL be saved through AssetService after a job terminal result is normalized.
- `cc-asset://asset/<assetId>` or documented equivalent is the renderer URL.
- Folder organization SHALL update folder relations, not leak absolute paths.
- Asset rename SHALL update `displayName` and `updatedAt` only; storage paths and safe URLs remain stable.
- Deleting a folder tree SHALL soft-delete folders and SHALL preserve referenced assets as recoverable tombstones when `force-tombstone` is requested.
- Asset create/update/trash/tombstone changes SHALL be emitted through `asset.changed` IPC events.
- The built-in starter image categories SHALL be role, scene, prop, and creature.
- Custom categories SHALL persist name, slug, color, icon, sort order, built-in flag, enabled state, and deleted state in SQLite.
- Asset category assignment SHALL preserve the underlying asset record and SHALL expose category IDs on renderer-safe `AssetRecord` values.
- Renderer upload progress SHALL be modeled as local multi-file import state with file index/count, current filename, percent complete, and last failed filename.
- No `asset.uploadProgress` IPC channel is introduced; renderer upload cards derive progress from sequential `asset.import` calls and `asset.changed` invalidation.
- Saved canvas graphs SHALL sync node asset selections and video reference inputs into `asset_references`.
- Completed media jobs SHALL sync their terminal asset result into `asset_references` with `refType = job`.
- The preload bridge SHALL expose a typed `asset.changed` subscription helper that returns an unsubscribe callback.
- Renderer asset refresh SHALL use IPC events plus query invalidation, not `setInterval`, asset polling loops, or TanStack Query `refetchInterval`.
- Renderer previews SHALL continue to use local `cc-asset://asset/<assetId>` safe URLs.
- Runtime execution payloads MAY use cloud URLs only through the workflow asset resolver. The resolver SHALL call the configured storage provider `query(s3Key)` to refresh or re-sign cloud access when storage is configured.
- Refreshed cloud URLs SHALL pass a host guard against the configured storage endpoint or `publicUrlPrefix`; failed refreshes, missing storage config, missing `s3Key`, and foreign hosts SHALL fall back to the local `cc-asset://` URL.
- MJ URL refresh/re-sign behavior is out of scope for the local Phase A migration.

## Errors

| Error class | Meaning |
| :--- | :--- |
| `asset_not_found` | Asset ID does not exist. |
| `asset_path_traversal` | Safe protocol or import path attempted traversal. |
| `asset_metadata_invalid` | Dimensions, duration, or media metadata failed validation. |
| `asset_unsupported_extension` | Import file extension is outside the local asset library allowlist. |
| `asset_reference_blocked` | Destructive action blocked by existing references. |
| `asset_display_name_required` | Rename request did not include a usable display name. |
| `asset_category_not_found` | Category ID does not exist or has been deleted. |
| `asset_category_name_required` | Category creation request did not include a usable name. |
| `asset_storage_unavailable` | App-controlled asset root cannot be read or written. |

## Permissions

- Importing files requires user-approved file access.
- Moving/renaming folders and renaming asset display labels are local library write actions.
- Trash/delete requires reference-integrity checks and may require `ask` if referenced.
- Renderer receives logical asset IDs and safe URLs only.

## Tests

- Unit: orientation classification for landscape, portrait, and square.
- Unit: imported image/audio metadata extracts dimensions, orientation, MIME, size, hash, and best-effort duration where feasible.
- Integration: generated bytes are stored with relative paths.
- Integration: unsupported import extensions are rejected before local copy or cloud upload.
- Integration: safe protocol rejects traversal.
- Integration: referenced asset trash either blocks or creates recoverable tombstone according to request mode.
- Integration: nested folder delete preserves references by tombstoning referenced assets and trashing unreferenced assets.
- Repository/IPC/Preload/UI: asset display rename preserves storage paths and safe URLs.
- Renderer: sidebar category settings edit custom category color/icon and disable custom categories through typed update APIs.
- Renderer: upload card shows multi-file progress from local import state without polling or a dedicated upload-progress IPC channel.
- Renderer: preview modal assigns and removes image categories through typed asset category APIs.
- Renderer: canvas asset panel filters by image category and inserts the same asset ID as image, character, scene, or selected video reference input.
- Main/Repository: saved canvas graph references and completed job asset results block safe asset deletion through SQLite `asset_references`.
- Main: workflow asset resolver refreshes configured cloud URLs, rejects foreign hosts, and falls back to local `cc-asset://` safe URLs.
- Renderer: `asset.changed` invalidates all-assets and asset-by-id queries without polling.
- Repository: starter categories, custom categories, tags, category assignment, category filtering, and disabled category filtering.
- IPC: category list/create/update/assign/remove are typed and use the repository boundary.
- Deep scan: renderer-facing asset records contain no absolute paths.
