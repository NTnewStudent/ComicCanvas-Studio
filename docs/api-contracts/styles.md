# Style Presets API Contract

## Purpose

Style presets provide project-level and node-level 画风 control for prompt
preview and generation jobs. The shared prompt function is the single rule for
both renderer preview and main-process runtime payload construction.

## Shared Contracts

- `StylePresetView`: renderer-safe style preset with `id`, `code`, `name`,
  `description`, `promptBefore`, `promptAfter`, `legacyPromptPreset`,
  `negativePrompt`, `coverAssetId`, `coverUrl`, `tags`, `enabled`,
  `sortOrder`, `createdAt`, and `updatedAt`.
- `StylePresetSaveInput`: create/update input. Secret values and absolute local
  paths are not valid fields.
- `StyleProjectDefaultRequest`: `{ workflowId, stylePresetId }` where
  `stylePresetId: null` clears the project default.

## Prompt Composition

`composeStyledPrompt(content, style)` is pure and deterministic:

- Trim `content`, `promptBefore`, and `promptAfter`.
- If `promptBefore` or `promptAfter` is present, join non-empty parts with
  `\n` in this order: `promptBefore`, content, `promptAfter`.
- If both before/after are empty, use `legacyPromptPreset` as
  `content + "\n\n画面风格：" + legacyPromptPreset`.
- If no style parts exist, return trimmed content unchanged.

The runtime and preview paths SHALL use this same shared function or a
byte-equivalent generated derivative.

## Effective Style Resolution

Node-level `stylePresetId` overrides the workflow project default. Missing or
disabled effective styles are recoverable run validation errors:

- `style_not_found`: selected style is unavailable.
- `style_disabled`: selected style exists but is disabled.

Save/load MUST remain non-destructive when a referenced style is missing or
disabled.

Deleting a style preset SHALL soft-delete/disable the preset and clear workflow
project defaults that point at that preset, while preserving existing node-level
`stylePresetId` graph data so stale-node validation can report the issue.

## IPC Channels

- `style.list`: list enabled and disabled style presets visible to settings or
  canvas surfaces.
- `style.save`: create or update a style preset.
- `style.delete`: soft-delete or disable a preset without mutating existing
  graph node data; workflow project default references to the deleted preset
  SHALL be cleared.
- `style.setProjectDefault`: set or clear a workflow's default style preset.
- `style.getProjectDefault`: read a workflow's current default style preset
  so renderer project selectors can initialize from persisted state.

Renderer-facing responses SHALL contain safe asset URLs only. They SHALL NOT
contain absolute local paths, API secrets, generated bytes, or provider-specific
temporary URLs.

## Evidence

Required verification before REQ-094 can be marked complete:

- Shared contract and prompt composition unit tests.
- Repository and IPC tests for style persistence and project default updates.
- Renderer style library, canvas style panel, and node/project selector tests.
- Desktop flow: select project style, override node style, run a stub job, and
  verify the job payload uses the expected styled prompt.
