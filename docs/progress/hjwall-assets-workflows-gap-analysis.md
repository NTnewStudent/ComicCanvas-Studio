# hjwall Assets + Workflows Gap Analysis

Date: 2026-06-27

Canonical spec: `specs/hjwall-assets-workflows-100-migration/`

This report is the Phase A migration inventory for hjwall `assets` and
`workflows/workflow-canvas`. LTM is not used. hjwall code is reference-only and
must not be copied into ComicCanvas.

## Migration Decision

1. **Phase A:** 100% migrate hjwall Assets + Workflows, including UI details.
2. **Phase B:** Add Agent capability on top of the completed manual workflow
   vocabulary and completed Tool/UI equivalence layer.
3. **Phase C:** Evolve the completed workflow editor into an infinite canvas.

Adaptation rule: hjwall role/scene/prop/creature asset resources become custom
image asset categories/tags in ComicCanvas. The local version ignores Seedance,
real-person, and liveness-auth flows.

## Current ToolRuntime Coverage

Existing built-in canvas tools:

| Tool ID | Current Scope | Gap For Phase A |
| :--- | :--- | :--- |
| `canvas.queryGraph` | Read current graph snapshot. | Needs workflow/version scoped reads and validation summaries. |
| `canvas.proposePlan` | Return a declarative plan draft. | Agent phase only; keep frozen during manual parity. |
| `canvas.createNode` | Create a node with supplied data. | Needs default data, layout, category/asset insertion semantics. |
| `canvas.connectNodes` | Add validated edge by matrix. | Needs semantic edge roles, duplicate handling, connect-to-create parity. |
| `canvas.updateNodeData` | Merge node data. | Needs rename, position, batch updates, asset reference side effects. |
| `canvas.deleteNode` | Delete node and attached edges. | Needs batch delete, reference cleanup/tombstone policy. |
| `canvas.runNode` | Enqueue current image/video-ish node jobs. | Needs migrated node actions, compile snapshots, output bindings. |

Missing Agent-ready tool/service coverage:

- Graph validate/save/load/version/import/export.
- Node duplicate/rename/position/batch mutation and default-data creation.
- Edge delete/update-role/connect-to-create and semantic ordering.
- Selection, layout, snippet save/insert, and project/template actions.
- Asset category CRUD, asset assignment, categorized insertion, and reference-safe delete.
- Style resolution, stale style validation, job recovery, media drop/edit, and output binding.

## Recommended Implementation Order

1. Finish Phase 0: keep this inventory, backlog, and task file in sync and
   enforce that with static tests.
2. Build Phase 1 asset foundations: category contracts, repository/migration,
   import metadata, and asset references.
3. Build Phase 1 UI: asset page parity, upload card, asset card/preview/batch
   mode, custom categories, and categorized insertion.
4. Move to workflow project/template parity only after asset references and
   categories are stable.
5. Keep Tool/UI equivalence active across every durable manual behavior so
   Agent expansion later calls the same semantics.

## Phase A Acceptance Model

- Engineering completion: implementation review plus automated tests.
- Automated smoke evidence: `phase-a-assets-workflows-smoke.test.ts` covers
  project create, canvas node add/save/reopen, imported categorized asset
  insertion, safe-delete blocking through synced references, snippet
  save/insert, and stub job terminal completion.
- Product acceptance: human desktop review rows in
  `docs/progress/human-desktop-review-checklist.md`.
- `HDR-PHASEA-001` is the final review gate. Phase A is not accepted until
  human review pass or explicit product deferral.
- Agent expansion remains deferred until Phase A manual asset/workflow parity
  and Tool/UI equivalence are engineering-complete.
