# hjwall Assets + Workflows 差距分析

日期：2026-06-27

标准规格：`specs/hjwall-assets-workflows-100-migration/`

本报告是 hjwall `assets` 与 `workflows/workflow-canvas` 的 Phase A 迁移清单。不使用 LTM。hjwall 代码仅作参考，不得直接复制进 ComicCanvas。

## 迁移决策

1. **Phase A：** 100% 迁移 hjwall Assets + Workflows，包括 UI 细节。
2. **Phase B：** 在已完成的手动 workflow 词汇表与已完成的 Tool/UI 等价层之上添加 Agent 能力。
3. **Phase C：** 将已完成的 workflow 编辑器演进为无限画布。

适配规则：hjwall 的角色/场景/道具/生物资产资源在 ComicCanvas 中变为自定义图片资产分类/标签。本地版本忽略 Seedance、真人、活体认证相关流程。

## 当前 ToolRuntime 覆盖情况

现有内置画布工具：

| Tool ID | 当前范围 | Phase A 差距 |
| :--- | :--- | :--- |
| `canvas.queryGraph` | 读取当前图快照。 | 需要 workflow/version 范围化读取与校验摘要。 |
| `canvas.proposePlan` | 返回声明式 plan 草案。 | 仅 Agent 阶段使用；手动对等期间保持冻结。 |
| `canvas.createNode` | 用给定数据创建节点。 | 需要默认数据、布局、分类/资产插入语义。 |
| `canvas.connectNodes` | 按矩阵添加已校验的边。 | 需要语义化边角色、重复处理、connect-to-create 对等。 |
| `canvas.updateNodeData` | 合并节点数据。 | 需要重命名、位置、批量更新、资产引用副作用。 |
| `canvas.deleteNode` | 删除节点及关联边。 | 需要批量删除、引用清理/tombstone 策略。 |
| `canvas.runNode` | 入队当前图片/视频类节点任务。 | 需要迁移后的节点动作、编译快照、输出绑定。 |

缺失的、面向 Agent 就绪的 tool/service 覆盖：

- 图的 validate/save/load/version/import/export。
- 节点的 duplicate/rename/position/批量变更以及默认数据创建。
- 边的 delete/update-role/connect-to-create 与语义化排序。
- 选区、布局、片段 save/insert、以及项目/模板操作。
- 资产分类 CRUD、资产归类、分类化插入、引用安全删除。
- 样式解析、过期样式校验、任务恢复、媒体拖放/编辑、输出绑定。

## 推荐实现顺序

1. 完成 Phase 0：保持本清单、backlog 与任务文件同步，并用静态测试强制约束。
2. 构建 Phase 1 资产基础设施：分类契约、仓储/迁移、导入元数据以及资产引用。
3. 构建 Phase 1 UI：资产页对等、上传卡片、资产卡片/预览/批量模式、自定义分类，以及分类化插入。
4. 只有在资产引用与分类稳定之后，才推进 workflow 项目/模板对等。
5. 在所有可持久保留的手动行为上保持 Tool/UI 等价性激活，以便后续 Agent 扩展调用相同的语义。

## Phase A 验收模型

- 工程完成：实现复核 + 自动化测试。
- 自动化冒烟证据：`phase-a-assets-workflows-smoke.test.ts` 覆盖项目创建、画布节点添加/保存/重新打开、导入的分类化资产插入、通过同步引用实现的安全删除拦截、片段保存/插入，以及占位任务的终态完成。
- 产品验收：`docs/progress/human-desktop-review-checklist.md` 中的人工桌面复核行。
- `HDR-PHASEA-001` 是最终复核关卡。在人工复核通过或产品方明确延后决定之前，Phase A 不算通过验收。
- Agent 扩展将持续推迟，直到 Phase A 的手动资产/workflow 对等以及 Tool/UI 等价性在工程上完成为止。
