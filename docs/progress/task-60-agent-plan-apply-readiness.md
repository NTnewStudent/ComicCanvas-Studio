## Current State

Task 60 is implemented behind `shared/agent-plan-apply.ts`. Renderer chat surfaces
auto-apply a ready `CanvasPlan` through the existing `applyPlan` +
`createCanvasPlanExecutionController` path when:

- `AGENT_PLAN_AUTO_APPLY_ENABLED` is true, and
- the user enables chat「自动执行」, or the selected agent has
  `triggerPolicy.autoRun = true`, and
- the plan `kind` is `plan` (not `clarify`).

Manual PlanCard apply remains available when auto-run is off.

**Product deferral (2026-07-05):** `HDR-PHASEA-001` remains Pending; product owner
deferred batched human acceptance until engineering queue completion. Task 60
closed on automated evidence (`tests/agent-plan-apply-run.test.ts` 8/8 with
related gate tests). HDR-050 / HDR-051 await the batch review session.

## Scope When Gate Opens

Task 60 SHALL implement Agent plan apply/run over the completed non-MJ workflow
vocabulary only:

- Nodes: text, image, video, imageConfigV2, videoConfigV2, character, scene,
  audio, videoCompose, superResolution, muxAudioVideo.
- Edges: promptOrder, imageOrder, imageRole, outputLink, reference, default.
- Run actions: imageRun, videoRun, textPolish.

MJ remains legacy-known but unavailable. Seedance/live-person flows and LTM are
out of scope.

## Required Reuse Points

When implementation starts, Task 60 SHALL reuse these existing surfaces rather
than creating parallel behavior:

- Plan validation and sanitization: `sanitizePlan`, shared node vocabulary,
  shared edge vocabulary, and graph validation.
- Manual apply semantics: renderer `applyPlan` behavior, one undo snapshot,
  deterministic placement, and dropped warning display.
- Manual run semantics: existing `canvas.runNode`, JobQueue, JobWorker, IPC
  terminal events, and CanvasJobPanel recovery display.
- Tool path: current `createCanvasTools` descriptors and ToolRuntime structured
  errors for graph mutations and run tickets.
- Durable services: existing asset/category, workflow project/template,
  snippet, style/model, storage, and job repository or IPC/service contracts.

No new Agent-only graph mutation, style resolution, asset resolution, run
enqueue, or storage path may bypass these surfaces.

## Must Not Implement

Task 60 MUST NOT:

- Start while `HDR-PHASEA-001` is Pending without explicit product deferral.
- Enable MJ planning, MJ add/run paths, MJ URL refresh, MJ multi-result parity,
  or MJ provider integration.
- Reintroduce Seedance/live-person flows.
- Use LTM for context, state, checkpoints, or evidence.
- Return generated bytes, asset URLs, absolute paths, or secrets from sync
  Agent/apply/run responses.
- Create a second connection matrix, style resolver, asset resolver, or job
  queue for Agent-only use.

## Minimum Verification When Implemented Later

When the gate opens and implementation begins, verification must include:

- Agent plan apply tests for migrated nodes and edges.
- Agent run serialization tests that prove terminal events advance steps in
  order and failures leave remaining steps manual.
- Dropped-warning and clarify tests for MJ, unsupported assets/styles/models,
  illegal edges, unavailable run actions, and executable strings.
- Tool/UI equivalence regression proving the Agent path uses the same durable
  semantics as manual UI for mutations and run tickets.
- `bun run typecheck` and `git diff --check`.
